## 推荐部署方式：GitHub + Vercel（无需本地 Node.js）

这是最简单的方式，不需要在本地安装任何开发环境！

### 步骤 1：安装 Git（如果还没有）
下载并安装 Git：https://git-scm.com/download/win

### 步骤 2：创建 GitHub 仓库并推送代码

打开 PowerShell，在项目目录运行：

```powershell
# 初始化 Git 仓库
git init

# 添加所有文件
git add .

# 提交代码
git commit -m "Initial commit: B站视频字幕提取工具"

# 在 GitHub 上创建新仓库后，添加远程仓库
# 将下面的 <your-username> 和 <your-repo-name> 替换为你的 GitHub 用户名和仓库名
git remote add origin https://github.com/<your-username>/<your-repo-name>.git

# 推送到 GitHub
git branch -M main
git push -u origin main
```

### 步骤 3：在 Vercel 上部署

1. 访问 Vercel 官网：https://vercel.com
2. 点击 "Sign Up"（如果没有账号）或 "Login"
3. 使用 GitHub 账号登录（推荐）
4. 登录后，点击 "New Project"
5. 选择 "Import Git Repository"
6. 找到你刚才推送的 GitHub 仓库
7. 点击 "Import"
8. Vercel 会自动检测项目配置：
   - Framework Preset: Vite
   - Build Command: `npm run build`
   - Output Directory: `dist`
   - Install Command: `npm install`
9. 点击 "Deploy"
10. 等待几分钟，部署完成！

### 步骤 4：获取访问链接

部署成功后，你会看到：
- 🎉 一个类似 `https://your-project-name.vercel.app` 的链接
- 这就是你的应用访问地址！

## 备选方式：本地部署（需要先安装 Node.js）

如果你想在本地测试或使用 CLI 部署：

### 1. 安装 Node.js
- 下载地址：https://nodejs.org/
- 推荐下载 LTS（长期支持）版本
- 安装后重启 PowerShell

### 2. 验证安装
```powershell
node --version
npm --version
```

### 3. 安装项目依赖
```powershell
npm install
```

### 4. 本地测试
```powershell
# 构建项目
npm run build

# 预览构建结果
npm run preview
```

### 5. 使用 Vercel CLI 部署
```powershell
# 安装 Vercel CLI
npm install -g vercel

# 登录
vercel login

# 部署（首次）
vercel

# 生产环境部署
vercel --prod
```

## 常见问题

### Q: 我没有 GitHub 账号怎么办？
A: 可以使用 GitLab 或 Bitbucket，Vercel 也支持这些平台。

### Q: 部署后如何更新代码？
A: 只需推送新代码到 GitHub，Vercel 会自动重新部署：
```powershell
git add .
git commit -m "更新说明"
git push
```

### Q: 如何查看部署日志？
A: 在 Vercel 仪表板中选择你的项目，点击最新的部署，可以查看详细日志。

### Q: 部署失败怎么办？
A: 
1. 查看 Vercel 部署日志中的错误信息
2. 确保所有文件都已正确推送到 GitHub
3. 检查 `package.json` 中的依赖是否正确
4. 如果构建失败，可能是依赖版本冲突，尝试删除 `package-lock.json` 重新部署

### Q: 如何配置自定义域名？
A: 在 Vercel 项目设置中：
1. 进入 Settings → Domains
2. 添加你的域名
3. 按照提示配置 DNS 记录（A 记录或 CNAME）
